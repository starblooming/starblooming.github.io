社媒自动化运营平台建设方案

建设阶段：一期（核心发布工作台）+ 二期（评论闭环扩展）

一、项目总览

本方案旨在搭建一套轻量、稳定、可落地的社媒自动化运营平台，一期聚焦「多账号内容发布工作台」，解决账号矩阵管理、素材统一管理、定时批量发布、任务可追溯等核心痛点；二期基于一期能力，扩展评论采集与智能回复闭环，兼顾运营效率与平台合规性。

二、一期建设：可运营发布工作台（核心重点）

2.1 核心目标

  1. 账号矩阵管理
    1. 支持多平台、多账号统一管理
    2. 账号采用独立Cookie隔离
    3. 账号登录状态失效可可视化展示
  2. 素材中心管理
    1. 素材与发布任务解耦，独立维护
    2. 支持视频、封面图及对应元数据管理
  3. 发布排期能力
    1. 支持立即发布、定时发布两种模式
    2. 支持批量选择账号，一键生成发布任务
  4. 任务运维管控
    1. 任务状态全流程流转：PENDING → RUNNING → SUCCESS/FAILED/NEED_LOGIN
    2. 支持失败任务重试、执行日志留存
  5. 系统稳定性保障
    1. 任务幂等设计，避免重复执行
    2. 账号环境完全隔离，防止串号风险
    3. 数据（数据库+文件）持久化存储，不丢失

2.2 核心技术实现

2.2.1 整体架构

  • 复用现有前端页面、后端API服务
  • 新增独立Worker节点，基于Playwright执行自动化操作
  • 数据存储：SQLite + 本地文件存储

2.2.2 关键运行策略

  • 执行模式：单Worker、账号串行执行，规避平台风控与串号问题
  • 任务调度：基于SQLite原子抢占任务，防止重复执行
  • 数据库优化：开启SQLite WAL模式，提升并发读写稳定性
  • 登录与发布：
    • 登录采用Playwright Headful模式
    • 每个账号使用独立运行目录与独立Cookie
    • 任务失败自动留存日志、异常截图

2.2.3 核心API范围

  • 账号模块：增删改查、状态查询
  • 素材模块：增删改查、元数据管理
  • 任务模块：增删改查、任务重试
  • 日志模块：执行日志查看

2.2.4 部署方案

  • 部署方式：Docker Compose一键部署
  • 服务组成：backend（后端） + worker（自动化执行）
  • 持久化：挂载4个独立持久化目录，保障数据不丢失

2.3 一期验收标准（3条核心）

  1. 运营全流程闭环

  2. 运营人员可独立完成：账号状态查看 → 素材上传 → 批量定时发布 → 任务进度追踪

  3. 故障可查可处理

  4. 任务失败展示失败原因+异常截图，支持重试；Cookie失效自动标记NEED_LOGIN

  5. 数据绝对可靠

  6. 服务重启/异常退出后，账号信息、素材文件、任务数据均完整保留，不丢失

三、二期建设：评论采集与回复闭环（非一期强制，可扩展）

3.1 核心目标

  1. 评论采集
    1. 优先覆盖自家账号旗下作品评论
    2. 支持增量采集、数据入库、检索与分配
  2. 评论回复
    1. 支持回复模板、商品信息快捷调用
    2. 平台不支持自动回复时，采用半自动模式：跳转原评论 + 一键复制话术/商品卡片
    3. 保证回复效率，同时遵守各平台规则

3.2 未来拓展方向

  1. OpenClaw/Agent 工作流编排

  2. 串联全流程：规则配置 → 评论采集 → 内容筛选 → 回复生成 → 人工审核 → 执行发布 → 结果回写

  3. 支持插件化扩展、执行日志可回放

  4. AI 智能评论处理

    1. 评论分类：用户意向、情绪、风险内容自动识别
    2. 自动摘要：提炼评论核心诉求
    3. 结合商品库生成个性化回复，输出后经审核发布

四、阶段总结

  1. 一期：以「发布工作台」为核心，实现多账号、多素材、多任务的标准化、自动化发布，保障稳定、可用、可运维。
  2. 二期：以「评论闭环」为延伸，实现从内容发布到用户互动的全链路运营，逐步接入AI与工作流编排，提升智能化水平。
  3. 技术底线：全程遵循账号隔离、任务幂等、数据持久化，兼顾自动化效率与平台合规安全。
